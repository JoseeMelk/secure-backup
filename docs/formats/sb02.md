# SB02 Format Specification

Status: Current

---

## Overview

SB02 extends SB01 by introducing embedded metadata support.

---

## Binary Layout

| Field | Size |
|---|---|
| MAGIC (`SB02`) | 4 bytes |
| Version | 1 byte |
| Metadata Length | 2 bytes |
| Metadata | Variable |
| Salt | 16 bytes |
| Nonce | 12 bytes |
| Authentication Tag | 16 bytes |
| Ciphertext | Variable |

---

## Metadata Schema

Example:

```json
{
  "filename": "photo.png"
}
```

---

## Notes

    * Metadata is stored as UTF-8 JSON
    * Metadata is not encrypted
    * SB02 remains memory-based internally
    * Designed for future parser compatibility

---

## Security Notes

Metadata may expose:

    * original filename
    * future optional attributes

Sensitive metadata should not be stored.
